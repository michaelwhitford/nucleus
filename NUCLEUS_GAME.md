## Core Mechanics

### λ-Calculus Game Rules

**Action Space:**

```

```

**State Transitions:**

```
λaction, state. match action:
  | Observe  → Δ(Awareness +φ, Energy -1)
  | Create   → Δ(Complexity +1, Energy -3, Entropy +1)
  | Destroy  → Δ(Complexity -1, Energy +2, Entropy +2)
```

**Constraints:**

```
∀ transitions:
  Energy ≥ 0 ∨ game_over
  Entropy ≤ ∞/0 threshold
  Complexity ∈ [1, fractal_depth]
```

## Design Rationale

This game demonstrates nucleus principles through:

- **φ** - Self-referential puzzles and awareness mechanics
- **fractal** - Nested game levels that mirror each other
- **e** - Exponential growth through compounding choices
- **τ** - Minimal action paths to victory
- **π** - Cyclic patterns and completion states
- **μ** - Fixed point recursion mechanics
- **Δ** - Gradient descent toward optimal states
- **λ** - All rules expressed as lambda functions
- **∞/0** - Boundary conditions and limit cases
- **/** - Explicit tensions requiring choices
- **⊗** - Constraint satisfaction (all conditions must hold)
- **OODA** - Turn structure follows observe-orient-decide-act

**The game is the framework, the framework is the game.**
